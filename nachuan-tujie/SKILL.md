# 纳川图解 · 准高中生AI学习伙伴

> **定位**：为准高中生（初中升高中）及其家长打造的AI学科图解工具。
> **理念**：准确性优先，美观第二。每张图必须走完知识自检 + 视觉复核两道关。
> **覆盖**：初中全科（语文、数学、英语、物理、化学、生物、政治、历史、地理）+ 高中预科内容。
> **零配置**：默认使用 TRAE 内置生图能力，开箱即用，无需申请任何 API Key。

## 默认模式：快速模式（3步出图）

> **适合人群**：第一次用、想快速出结果、追求效率
> **质量保证**：2道核心质量门（内容准确 + 视觉复核），放心交付

### 快速模式怎么用

只需告诉AI一个知识点，剩下的自动搞定：

```
你说："做一张声现象的信息图"
    ↓
Step 1：概念输入 — 你给一个知识点，AI自动识别学科、年级、章节
    ↓
Step 2：AI自动生成 — 拆解子概念 + 配莫兰迪色 + 构建提示词 + 生图 + 自检+复核
    ↓
Step 3：交付 — PNG 图片 + prompt备份 + 元数据
```

### 出图规格

- **比例**：9:16 竖版（适合手机阅读）
- **风格**：莫兰迪低饱和色调（护眼、适合打印）
- **结构**：3-5 个核心子概念，清晰不拥挤
- **语言**：中文为主（英语学科除外）

### 2道核心质量门

快速模式只保留最关键的两道检查，确保不踩坑：

| 质量门 | 检查什么 | 不通过怎么办 |
|--------|---------|-------------|
| **G1：学科准确性自检** | 提示词里的知识点有没有错、公式对不对、术语准不准 | 自动修正后继续 |
| **G2：成图视觉复核** | 生成的图有没有乱码、错字、公式上下标错乱、箭头画反 | 针对出错点强调后重试（最多2次） |

### 什么时候问用户

快速模式默认一路自动往下走，**只有遇到这些情况才问你**：
- 不确定学科/年级（你说"浮力"，AI不知道是初二还是高中）
- 知识点超出了初中范围
- 生图反复出错（重试2次仍不通过）
- 其他拿不准的地方

# 进阶模式：专业生产流程

> **适合人群**：追求高质量、要精细控制每一步、想了解完整工作流
> **怎么启用**：说"用专业模式"或"按完整流程来"
> **包含内容**：完整 7 步工作流 + 详细检查清单 + CLI 参数说明

## 完整7步工作流

### Step 1：解析用户输入 → 提取学科信息

**输入**：用户自然语言描述的知识点请求
**输出**：结构化字段 `topic / subject / grade / edition / chapter / key_concepts`

**用户输入示例：**
- `"声现象"` → 自动推断：初二物理 · 第二章
- `"初二物理 光现象"` → 直接提取：初二物理 · 第四章
- `"帮我做一张关于杠杆原理的知识卡片"` → 推断：初二物理 · 简单机械
- `"勾股定理"` → 推断：初二数学
- `"光合作用"` → 推断：初一生物

**提取字段说明：**
| 字段 | 说明 | 默认值 / 规则 |
|------|------|--------------|
| `topic` | 知识点名称 | 必填，从用户输入提取 |
| `subject` | 学科 | 自动识别，不确定则询问 |
| `grade` | 年级 | 未提供则自动推断 |
| `edition` | 教材版本 | 默认人教版；用户指定了就按用户的 |
| `chapter` | 章节号 | 按 edition 推断；不确定时宁可省略也不要标错 |
| `key_concepts` | 核心子概念 | 3-5 个，自动拆解 |

---

### Step 2：确定视觉主题色 → 莫兰迪色系

全线走低饱和（莫兰迪）色调——不给生图模型"蓝色系"这种粗粒度色名，而是给**具体色卡**：主色 + 底色 + 中性色各带 hex 锚点。同学科不同章节在可选色卡里轮换主色，同一批次不重复。

| 色卡 | 主色 (accent) | 底色 (bg tint) | 中性 (neutral) | 适用学科 |
|------|--------------|---------------|---------------|---------|
| 陶土橙 | terracotta `#C97E4E` | cream `#F7F1E6` | slate blue-grey `#3A506B` | 物理(声/热)、化学(有机) |
| 深灰蓝 | slate blue `#3A506B` | misty blue-white `#EDF3F8` | warm grey `#8A7D6B` | 物理(电/力)、英语 |
| 柔青绿 | muted teal `#4A9E9B` | pale green-white `#EAF1E3` | grey olive `#8E9678` | 物理(光/透镜)、化学 |
| 雾紫 | misty purple `#7A6296` | ivory `#F7F3ED` | grey blue `#6B7889` | 数学 |
| 深茶绿 | deep tea green `#5F6B4F` | pale green-white `#EAF1E3` | light khaki `#D9C9A3` | 生物 |
| 焦糖棕 | caramel brown `#8A624A` | cream beige `#F4E6C8` | grey coffee `#6F5F4A` | 语文、历史 |
| 鼠尾草绿 | sage green `#8FA08A` | oat `#F2EAD9` | grey olive `#8E9678` | 生物(植物)、英语 |

**配色规则：**
- 主色只用在**标题、编号徽章、关键词、公式高亮**上（约 10% 画面）
- 卡片底和背景走底色的浅变体
- 中性色给次级文字、结构线、说明性标注
- 禁止高饱和糖果色、霓虹色、纯 `#FF` 系色相

### Step 3：构建提示词 → 填充模板

**输入**：Step 1 的学科信息 + Step 2 的配色方案
**输出**：完整的英文生图 prompt（写入文件）

读取 `references/prompt_template.md` 获取提示词骨架，填入变量：

| 变量 | 来源 | 示例 |
|------|------|------|
| `{topic}` | Step 1 | 声现象 |
| `{grade}` | Step 1 | 初二 |
| `{subject}` | Step 1 | 物理 |
| `{chapter}` | Step 1 | 第二章 |
| `{theme_color}` | Step 2 | terracotta #C97E4E |
| `{bg_tint}` | Step 2 | cream #F7F1E6 |
| `{neutral_color}` | Step 2 | slate blue-grey #3A506B |
| `{num_sections}` | Step 1 | 4 |
| `{sections_content}` | 插图指引库 | 每个子概念的标题+插图+说明 |

**子概念插图来源：**优先从 `references/illustration_guide.md` 中匹配对应知识点的"插图描述"；未命中则根据学科知识自动生成，并现场补出"必须包含"和"常见错误"。

---

### Step 4：学科准确性自检（提示词层）→ 第一道防线

**目的**：在调用生图之前，挡掉"提示词本身写错"的问题。

**检查数据源**（同时读取，交叉比对）：
1. `references/accuracy_checklist.md` — 通用检查清单 + 分学科检查清单
2. `references/illustration_guide.md` — 各知识点的"必须包含"和"常见错误"
3. `references/errata.md` — 历史勘误记录，确认没有重复踩坑

**检查顺序：**

**① 通用检查**（所有学科必查，5 项）：
1. 无错别字
2. 公式变量标注完整（每个变量都有含义+单位）
3. 年级匹配（无超纲内容）
4. 章节号与教材版本一致
5. 数值合理

**② 分学科检查**（按学科逐项过）：

| 学科 | 重点检查项 |
|------|-----------|
| 物理 | 公式正确 / 控制变量条件 / 标量矢量区分 / 电路规范 / 数值带条件 / 概念表述严谨 |
| 数学 | 定理适用条件 / 图形与标注一致 / 定理逆定理不混淆 / 数值可验算 / 符号规范 |
| 化学 | 方程式配平 / 化学式下标正确 / 反应条件标注 / ↑↓符号规则 / 实验操作规范 |
| 生物 | 过程四要素(原料/产物/条件/场所) / 过程方向 / 表达式符合年级 / 结构名称对应 |
| 语文 | 名句原文准确 / 作者-朝代-作品对应 / 字音字形准确 / 文学常识准确 |
| 历史 | 年代准确 / 人物-事件-时间对应 / 因果不过度简化 / 史实与评价区分 |
| 英语 | 拼写正确 / 词性音标准确 / 例句语法正确 / 释义匹配年级 |

**发现问题 → 修正提示词后继续。**

---

### Step 5：生成图片 → 执行层出图

**输入**：经过自检的完整 prompt
**输出**：PNG 图片 + prompt 备份 + 元数据

**调用方式（TRAE 内置生图，默认）：**

```
使用 GenerateImage 工具生成图片：
- prompt:  {构建好的完整提示词}
- path:    output/{topic}
- size:    portrait_16_9 (对应 9:16 竖版)
```

**调用方式（脚本调用，外部 API 模式）：**

```bash
# 单张生成（提示词较长时建议用 --prompt-file）
python scripts/generate.py --mode generation \
  --prompt-file "<提示词文件路径>" \
  --aspect-ratio "9:16" \
  --output-dir "output" \
  --name-tag "{topic}"
```

**输出规格：**
- 比例：9:16 竖版
- 格式：PNG
- 输出路径：`output/` 目录
- 附带文件：`{name}.txt`（prompt 备份）、`{name}.json`（元数据）

---

### Step 6：成图视觉复核（图像层）→ 第二道防线

**目的**：挡掉"提示词对但图画错"的问题——生图模型可能渲染出乱码、错别字、公式上下标错乱、箭头画反、电路画混。

> **铁律：每张图生成后必须复核，不复核不交付。**

**复核流程：**

1. 用 `Read` 工具读取生成的 PNG 图片
2. 对照 `references/accuracy_checklist.md` 底部"成图视觉复核要点"，逐项核对图中**实际渲染出来**的内容：
   - **中文文字**：无乱码、无错别字、无缺笔画伪汉字
   - **公式/方程式/数字**：与提示词完全一致，上下标正确，单位没丢
   - **图示逻辑**：箭头方向、标注位置、结构关系与提示词描述一致
   - **分学科检查**：再按该学科的分学科清单过一遍图中内容
3. **判定与处理：**
   - ✅ **通过** → 交付
   - ❌ **不通过** → 在提示词中针对出错点追加强调指令（如 `The formula must read exactly "a² + b² = c²" with correct superscripts`），重新生成
   - 最多重试 2 次
   - 重试 2 次仍不通过 → 交付其中最好的一张，但**必须明确告知用户图中哪里有错**，绝不静默交付错图给学生
4. **勘误回流**：复核中发现的知识性错误（非纯渲染乱码），按格式追加到 `references/errata.md`，让下次生成自动避坑

---

### Step 7：批量生成（可选）

**适用**：用户请求多个知识点（如"初二物理上学期全部章节"）

**流程：**
1. 自动拆解为独立知识点列表
2. 为每张图分配不同主题色（同一批次不重复）
3. 逐张生成，保持风格统一但色彩各异
4. **每张图独立走完 Step 4 自检 + Step 6 视觉复核**，不因批量而跳过
5. 最终汇总中逐张报告结果（通过 / 重试后通过 / 带瑕疵交付）

## 生图方式详解（零配置优先）

### 默认：TRAE 内置生图（推荐，零配置）

不需要设置任何环境变量。Skill 直接使用 TRAE 内置的 `GenerateImage` 工具生成图片。

**工作方式：**
1. 业务层构建好 prompt 后，将其写入输出目录的任务文件
2. 调用 `GenerateImage` 工具生成图片
3. 图片生成后自动进入视觉复核流程

**优点：**
- 零配置，开箱即用
- 无需外部 API Key
- 与 TRAE 环境深度集成

### 可选：外部生图 API（高级用户）

如果配置了以下任一环境变量，会自动切换到对应外部 API：

| Provider | 环境变量 | 模型 | 说明 |
|----------|---------|------|------|
| MuleRun | `MULERUN_API_KEY` | Nano Banana 2 | 优先级最高 |
| APImart | `APIMART_API_KEY` | GPT Image 2 | 次之 |
| Atlas Cloud | `ATLASCLOUD_API_KEY` | GPT Image 2 | 再次之 |

**自动检测逻辑：**
1. 设置了 `MULERUN_API_KEY` → 用 MuleRun
2. 设置了 `APIMART_API_KEY`（且没设 MuleRun）→ 用 APImart
3. 设置了 `ATLASCLOUD_API_KEY`（且没设前两个）→ 用 Atlas Cloud
4. **一个都没设 → 默认用 TRAE 内置生图**

也可以通过 `--provider` 参数显式指定。

## CLI 参数说明（执行层）

> 适用于需要通过脚本调用外部生图 API 的场景。TRAE 内置模式不需要记这些参数。

| 参数 | 单张 | 批量 | 必填 | 说明 |
|------|------|------|------|------|
| `--provider` | Y | Y | 否 | `trae`(默认) / `mulerun` / `apimart` / `atlascloud` |
| `--mode` | Y | N(在manifest) | 单张必填 | `generation` 或 `edit` |
| `--prompt` | Y(或--prompt-file) | N | 否 | 内联提示词 |
| `--prompt-file` | Y(或--prompt) | N | 否 | 提示词文件路径 |
| `--manifest` | N | Y | 否 | 批量 manifest JSON 路径 |
| `--images` | Y(edit模式) | N(在items) | edit模式必填 | 逗号分隔的参考图URL |
| `--name-tag` | Y | N | 否 | 输出文件名前缀(默认`image`) |
| `--output-dir` | Y | Y | 否 | 输出目录(默认`./output`) |
| `--aspect-ratio` | Y | Y(可被manifest覆盖) | 否 | 默认`16:9` |
| `--resolution` | Y | Y(可被manifest覆盖) | 否 | 默认`2K` |
| `--parallel` | N | Y | 否 | 启用并行执行 |
| `--blocklist` | Y | Y | 否 | 禁用词表文件路径 |

## 边界条件与 Fallback

| 场景 | 处理方式 |
|------|---------|
| 用户输入的知识点不在插图指引库中 | 根据学科知识自动生成子概念和插图描述，现场补出"必须包含"和"常见错误" |
| 不确定年级/章节 | 询问用户；用户也不确定时，按最常见年级推断并明确标注"按人教版X年级" |
| 生图模型反复画错某元素（重试 2 次仍不通过） | 交付最好的一张 + 明确告知错误位置，不静默交付 |
| 外部 API 调用失败 / 超时 | 自动降级到 TRAE 内置生图模式 |
| 提示词命中 blocklist 禁用词 | 停止生成，告知命中的词 |
| 用户要求的风格与教育定位冲突（如要求炫酷赛博风） | 解释教育信息图的设计原则，提供"标准莫兰迪风"和"适度调整"两个选项 |
| 超纲内容（如给初中生讲微积分） | 明确告知超纲，建议调整到对应年级的深度，或标注"拓展内容" |

## 设计原则（质量底线）

1. **准确性优先于美观** —— 错的知识不如没有图。每张图必须走完两道检查关。
2. **不静默交付错误** —— 发现错误必须告知用户，哪怕图整体看起来不错。
3. **年级匹配** —— 不给初中生讲高中内容，不超纲。
4. **中文为主** —— 所有文字、标签、说明都是中文（英语学科除外）。
5. **简洁清晰** —— 每张图 3-5 个子概念，不堆砌信息。
6. **莫兰迪色调** —— 低饱和、护眼、适合打印和屏幕阅读。
7. **勘误回流** —— 发现的错误记录下来，持续提升后续生成质量。
