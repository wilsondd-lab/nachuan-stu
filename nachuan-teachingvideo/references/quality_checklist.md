# 质量检查清单 · Quality Checklist

> 从脚本到渲染的全流程质量保证。
> 质量门不是"可以参考"，而是"必须检查并通过"。

---

## 质量门总览

| 质量门 | 阶段 | 检查项 | 严重程度 |
|--------|------|--------|---------|
| G1 | 脚本完成后 | 内容准确性 + 结构完整性 | 高 |
| G2 | 分镜完成后 | 五维分镜 + 动效有效性 | 中 |
| G3 | 场景编排完成后 | 视觉规范 + 音画同步 | 中 |
| G4 | 渲染完成后 | 渲染质量 + 最终 QA | 高 |

**快速模式**：仅保留 G1 + G4 两道核心质量门
**专业模式**：全部 4 道质量门

---

## G1：脚本质量门（内容准确 + 结构完整）

### G1-1 知识点准确性 🔴 必须通过

| 检查项 | 通过标准 | 未通过处理 |
|--------|---------|-----------|
| 核心定义 | 与教材一致，术语规范 | 查证后修正 |
| 公式符号 | 公式正确，符号标准，单位无误 | 重新核对 |
| 常量数值 | 物理/化学常数准确（如声速 340m/s、g=9.8） | 查证后修正 |
| 原理描述 | 因果关系正确，逻辑严谨 | 修正表述 |

**检查方法**：
- 搜索教材原文或权威来源
- 交叉验证 2 个以上来源
- 特别注意单位和数量级

---

### G1-2 结构完整性 🟡 重要

| 检查项 | 通过标准 | 未通过处理 |
|--------|---------|-----------|
| 7 段完整 | 标题 + 5 个子概念 + 总结，不缺段 | 补全缺失段落 |
| 逻辑递进 | 子概念按认知顺序排列，层层深入 | 调整顺序 |
| 钩子有效 | 首段制造信息缺口或认知冲突 | 重写钩子 |
| 首尾呼应 | 结尾回答开头提出的问题 | 调整结尾 |
| 过渡自然 | 段间有逻辑连接，不是生硬跳转 | 加过渡句 |

---

### G1-3 旁白质量 🟡 重要

| 检查项 | 通过标准 | 未通过处理 |
|--------|---------|-----------|
| 字数规范 | 每段 30-50 字（±10 字以内） | 精简或扩充 |
| 口语化 | 像老师讲解，不是念课本 | 改写为口语 |
| 不读公式 | 公式用口语解释，不直接念符号 | 改写读法 |
| 数字读法 | 所有数字写成读法（"3×10⁸"→"三亿"） | 补全读法 |
| 无口头禅 | 不用"然后然后""接下来我们看" | 删除或替换 |

---

### G1-4 概念密度 🟢 建议

| 检查项 | 通过标准 | 未通过处理 |
|--------|---------|-----------|
| 新概念数量 | 全片 ≤ 5 个新概念 | 砍掉超出内容，留到下一期 |
| 单段新概念 | 每段 ≤ 1 个新概念 | 拆分到多段 |
| 讲解时间 | 每个新概念有 ≥ 10s 讲解时间 | 延长对应段落 |

---

## G2：分镜质量门（五维分镜 + 动效有效）

### G2-1 五维完整性 🟡 重要

每个场景必须明确以下 5 个维度：

| 维度 | 检查要点 |
|------|---------|
| **Subject 主题** | 画面核心主体明确，和旁白讲的是同一个东西 |
| **Motion 动效** | 入场方式、运动轨迹、关键时序清晰 |
| **Scene 场景** | 布局、叠加层（chip/callout/ghost）完整 |
| **Spatial 空间** | 视觉焦点明确，元素分层合理，留白充足 |
| **Camera 镜头** | 景别明确（教学视频多为固定镜头，也要写清楚） |

---

### G2-2 动效有效性 🟡 重要

| 检查项 | 通过标准 | 未通过处理 |
|--------|---------|-----------|
| 服务教学 | 每个动效都服务于知识点讲解，不是装饰 | 删除无效动效 |
| 讲解顺序 | 动画顺序和自然讲解顺序一致（像黑板上讲题） | 调整时序 |
| 多样性 | 每场景 ≥ 3 种 ease、≥ 2 个入场方向 | 重新编排 |
| 节奏合理 | 不是所有元素同时出现，有 stagger 错峰 | 加 stagger |
| 无干扰 | 没有分散注意力的多余动效 | 删除冗余动效 |

---

### G2-3 信息密度 🟢 建议

| 检查项 | 通过标准 |
|--------|---------|
| 每场景信息块 | ≤ 3 个（示意图 + 卡片 + 标注） |
| 文字数量 | 卡片文字 ≤ 4 行，每行 ≤ 15 字 |
| 视觉层次 | 主次分明，一眼能看出重点 |

---

### G2-4 转场合理性 🟢 建议

| 检查项 | 通过标准 |
|--------|---------|
| 转场种类 | 全片 ≤ 3 种转场，以模糊叠化为主 |
| push-up 使用 | 仅用于有"章节推进"语义的地方，全片 ≤ 2 次 |
| 转场时长 | 统一 0.5s 左右，不要忽长忽短 |

---

## G3：场景编排质量门（视觉规范 + 音画同步）

### G3-1 设计规范一致性 🟡 重要

| 检查项 | 通过标准 | 未通过处理 |
|--------|---------|-----------|
| 配色统一 | 使用同一套调色板，主色不超过 2 种 | 统一配色 |
| 字体一致 | 标题用宋体粗体，正文用黑体 | 统一字体 |
| 组件规范 | chip、card、callout 样式统一 | 调整样式 |
| 间距系统 | 遵循 8pt 网格，间距不随意 | 调整间距 |

---

### G3-2 音画同步 🔴 必须通过

| 检查项 | 通过标准 | 未通过处理 |
|--------|---------|-----------|
| 时空同步 | 讲到哪个元素，哪个元素入场 | 调整 offset 时间 |
| 节奏匹配 | 示意图先画 → 标注弹出 → 卡片滑入（旁白讲到要点时） | 调整时序 |
| 不提前剧透 | 关键信息不要在旁白讲到之前就出现 | 延后入场 |
| 不滞后 | 旁白讲完了元素才出现 = 不同步 | 提前入场 |

**检查方法**：
- 关闭声音看画面 → 能看懂大概在讲什么吗？
- 只听声音不看画面 → 能跟上节奏吗？
- 音画一起看 → 讲到哪画面到哪吗？

---

### G3-3 可读性与安全区 🟡 重要

| 检查项 | 通过标准 |
|--------|---------|
| 安全边距 | 所有重要内容距离边缘 ≥ 80px |
| 字幕安全 | 字幕在底部安全区，不被裁切 |
| 字号达标 | 正文 ≥ 28px，标题 ≥ 48px |
| 对比度 | 文字与背景对比度 ≥ 4.5:1（正文）/ 3:1（大标题） |
| 无溢出 | 文字不超出容器，不被裁切 |

---

### G3-4 停留帧规范 🟢 建议

| 检查项 | 通过标准 |
|--------|---------|
| 停留时间 | 关键信息停留 1.5-2.5s（给理解和记忆时间） |
| 环境动效 | 停留期间有一种细微环境动效（呼吸/漂移/波形） |
| 一场景一种 | 环境动效一场景一种，不要每个元素都动 |

---

## G4：最终 QA（渲染质量 + 交付标准）

### G4-1 渲染质量 🔴 必须通过

| 检查项 | 通过标准 | 未通过处理 |
|--------|---------|-----------|
| 渲染成功 | MP4 文件存在，能正常播放 | 检查渲染日志 |
| 无花屏 | 没有黑屏、白屏、色块、闪烁 | 检查对应场景 HTML |
| 分辨率帧率 | 1920×1080，30fps 正常 | 检查渲染配置 |
| 文件大小 | 60-90s 视频约 5-15MB 合理 | 过大检查压缩设置 |
| lint 通过 | 0 error（warning 可接受） | 修复 lint 错误 |

---

### G4-2 字幕质量 🔴 必须通过

| 检查项 | 通过标准 | 未通过处理 |
|--------|---------|-----------|
| 同步准确 | 字幕出现时间和配音一致，误差 ≤ 0.2s | 调整时间轴 |
| 清晰可读 | 字号够大，对比度够，底部安全区 | 调整字幕样式 |
| 无错别字 | 字幕文字和旁白一致，无错字 | 修正文字 |
| 完整覆盖 | 每段旁白都有对应字幕 | 补全缺失字幕 |

---

### G4-3 音频质量 🟡 重要

| 检查项 | 通过标准 | 未通过处理 |
|--------|---------|-----------|
| 清晰流畅 | 旁白清晰，无明显机器味 | 换更好的 TTS 引擎 |
| 音量适中 | 音量在 -16 到 -12 LUFS 之间 | 调整音量 |
| 无爆音 | 没有削波、爆音、杂音 | 重新生成音频 |
| 各段一致 | 各段音量一致，没有一段大声一段小声 | 统一音量 |

---

### G4-4 视觉完整性 🟡 重要

完整看一遍视频，检查：

- [ ] 7 个场景都正常显示，没有空场景
- [ ] 配色整体统一，没有突兀的颜色
- [ ] 所有文字清晰可读，没有溢出
- [ ] 动效流畅，没有卡顿或跳变
- [ ] 转场自然，没有硬切（除非设计需要）
- [ ] 蒙太奇图 7 格都正常

---

### G4-5 交付标准 🔴 必须通过

**严重问题 = 0** 才能交付：
- 知识点错误
- 字幕不同步
- 渲染花屏
- 音频不可听
- 文字溢出/看不清

**轻微问题 ≤ 2 个** 可交付（记录后交付，或快速修复）：
- 某个元素位置略偏
- 动效节奏可优化
- 个别字发音略怪

---

## 快速检查流程（快速模式）

只有 5 分钟检查时间时，按这个顺序：

1. **看蒙太奇图**（1 分钟）：7 个场景都正常吗？配色统一吗？
2. **看开头 10 秒**（1 分钟）：钩子有效吗？画面清晰吗？
3. **看结尾 10 秒**（1 分钟）：总结到位吗？首尾呼应吗？
4. **抽查中间一段**（2 分钟）：音画同步吗？内容准确吗？

5 分钟能发现 80% 的严重问题。

---

## 常见问题快速定位

| 现象 | 可能原因 | 排查方向 |
|------|---------|---------|
| 渲染黑屏/白屏 | 根元素尺寸问题 / 背景在根上 | 检查 root 的 width/height，背景放在 scene 上 |
| 动效不动 | GSAP 没加载 / 时间轴没注册 | 检查 CDN 链接，检查 `window.__timelines` 注册 |
| 字幕不同步 | 音频时长估算不准 / offset 没调好 | 重新跑 TTS，同步时间轴 |
| 文字发糊 | 字体没声明 / 系统缺字体 | 加 `@font-face`，用系统字体 |
| 转场生硬 | 转场代码有问题 / 时间不对 | 检查 transition 类型和 timing |
| 画面挤 | 信息密度太高 / 间距太小 | 减少内容，加大间距 |
