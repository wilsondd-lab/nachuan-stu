## 一、四大核心时序原则

所有动效设计都围绕这四条原则展开。它们是让动画"看起来专业"的基础。

### 1. 缓入缓出（Ease In / Ease Out）

> 没有任何物体会瞬间启动或瞬间停止。所有运动都有加速和减速的过程。

**为什么重要：**
- 线性运动感觉机械、僵硬、"像电脑做的"
- 真实世界中，物体需要时间加速和减速
- 缓动让观众的眼睛能平滑跟随

**应用到教学视频：**
- 所有入场动画用 `ease out`（慢→快→慢，出场瞬间快、落位柔）
- 所有退场动画用 `ease in`
- 场景内的位置移动用 `ease in out`
- **只有 opacity 和颜色变化可以用 linear**（人眼对透明度的线性变化不敏感）

### 2. 预备动作（Anticipation）

> 在主要动作开始前，先有一个反向的小动作。就像跳远前要先下蹲、出拳前要先收拳。

**为什么重要：**
- 让观众"预知"接下来会发生什么，减少突兀感
- 增加动作的力量感和重量感
- 引导注意力

**应用到教学视频：**
- 重要元素入场前，可以先微微向反方向动一下再弹出来（`back.out` ease 的一部分）
- 卡片从右侧滑入前，可以先稍微右移再向左进入
- 不是所有元素都需要预备动作 — 只用于需要强调的关键元素（大数字、核心公式、关键示意图）

### 3. 超出回弹（Overshoot）

> 物体到达目标位置后，会稍微超过一点，再弹回来稳定下来。

**为什么重要：**
- 增加弹性和活力感
- 让入场有"落点"的感觉，而不是突然停住
- 适合表现轻松、鼓励的语气

**应用到教学视频：**
- 标题入场用 `back.out(1.5)` — 有轻微回弹
- 大数字/关键公式用 `back.out(1.6)` — 强调感更强
- 卡片和标签用 `power3.out` — 不回弹，更稳重
- **注意**：数学/物理等严肃主题回弹量要小（`back.out(1.2)`），避免"太跳"

### 4. 分层错峰（Staging / Stagger）

> 同一时间只有一个东西在动。多个元素入场时，依次错开，不要同时全部出现。

**为什么重要：**
- 观众的注意力一次只能聚焦在一处
- 同时动太多东西 = 什么都没看清
- 错峰入场有节奏感，像音乐的节拍

**应用到教学视频：**
- 多个 pill / badge 入场时，stagger 0.09-0.12 秒
- 示意图的多个部件，按逻辑顺序依次画出
- 卡片的多行文字，逐行淡入（stagger 0.08s）
- 每场景 ≥2 个入场方向 — 全从一个方向来 = 没编排

**错峰的节奏：**
- 快节奏（列表/标签）：stagger 0.08-0.12s
- 中节奏（卡片/图形）：stagger 0.15-0.25s
- 慢节奏（重要揭示）：stagger 0.3-0.5s

## 二、缓动曲线参考表

### GSAP Ease 速查

| Ease | 视觉感受 | 适用场景 | 不适用 |
|---|---|---|---|
| `power1.out` | 柔和，几乎是线性的尾段 | 次要元素淡出、颜色变化 | 主要入场 |
| `power2.out` | 标准，专业感 | 卡片滑入、标签出现 | 太普通，缺个性 |
| `power3.out` | 有力，干脆利落 | 主标题、重要卡片 | 太硬，缺柔度 |
| `power4.out` | 爆发力强，猛 | 超大标题、冲击性数字 | 太猛，不适合教学 |
| `back.out(1.2)` | 微弹，精致 | 小标题、图标 | — |
| `back.out(1.5)` | 明显回弹，活泼 | 主标题、徽章、数字 | 严肃主题慎用 |
| `back.out(2.0)` | 弹跳强，卡通感 | 趣味性强的内容 | 教学视频基本不用 |
| `sine.out` | 最柔和，丝滑 | 环境动效（呼吸、漂移） | 需要力度的入场 |
| `expo.out` | 极快启动，极慢收尾 | 大型转场、全屏揭示 | 太戏剧化 |

### 教学视频推荐组合

每场景至少 3 种 ease，形成节奏对比：

| 元素 | 推荐 Ease | 时长 | 理由 |
|---|---|---|---|
| 大标题 | `back.out(1.5)` | 0.7-0.8s | 有活力，吸引注意 |
| SVG 描边入场 | `power2.inOut` | 0.8-1.2s | 均匀绘制，像手绘 |
| 卡片 | `power3.out` | 0.6-0.7s | 干脆，不拖泥带水 |
| callout 标注 | `back.out(1.3)` | 0.4-0.5s | 弹出感，"指出来"的意味 |
| 标签/小标题 | `power2.out` | 0.4-0.5s | 标准，不抢戏 |
| 环境动效（呼吸） | `sine.inOut` | 0.8-1.2s | 柔和循环，不突兀 |
| 淡入（opacity） | `sine.out` | 0.4-0.6s | 柔和，不像动画 |

### 入场方向的多样性

每场景至少 2 个入场方向，避免单调：

- **从下往上**（y: 正数 → 0）：最自然的"升起"感，适合标题、主要内容
- **从上往下**（y: 负数 → 0）："落下"感，适合总结、结论
- **从右往左**（x: 正数 → 0）："推进"感，适合卡片、补充信息
- **从左往右**（x: 负数 → 0）："展开"感，适合流程图、时间线
- **缩放进入**（scale: 小 → 大）："聚焦"感，适合关键数字、核心公式
- **旋转进入**（rotation）：特殊场景慎用，容易分散注意力

## 三、停留帧规范（Hold Frames）

> 动画结束后，不要立刻开始下一个。让画面"停一下"，给观众时间消化。

### 停留时长参考

| 场景 | 停留时长 | 理由 |
|---|---|---|
| 关键信息（公式、大数字） | 1.5-2.5s | 需要时间理解和记忆 |
| 示意图绘制完成后 | 1.0-1.5s | 让视觉印象沉淀 |
| 卡片文字出现后 | 0.8-1.2s | 读完 2-3 行文字的时间 |
| 转场前的最后一帧 | 0.3-0.5s | 让当前场景完整呈现后再切 |
| 场景开头（转场后） | 0.2-0.3s | 转场结束后稍微停一下再开始入场 |
| 快速信息（标签、编号） | 0.5-0.8s | 短平快，不拖节奏 |

### "呼吸感" — 停留时的微动效

完全静止的画面会显得"死"。在停留期间加一个非常细微的环境动效，
既不分散注意力，又保持画面活力：

| 动效类型 | 参数建议 | 适用元素 |
|---|---|---|
| 轻微呼吸（scale 脉动） | scale 1.0 → 1.03，0.8s 周期，sine.inOut | 大数字、核心图标 |
| 缓慢漂移（y 浮动） | y ±6px，1.5s 周期，sine.inOut | 主题图标、示意图主体 |
| 波形流动 | 恒定速度 220px/s | 波形、扩散环 |
| 透明度脉动 | opacity 0.7 → 1.0，1s 周期 | 高亮元素、能量标识 |

**原则：**
- 环境动效的幅度一定要小 — 远看几乎注意不到，细看才有
- 一场景一种环境动效就够了，不要每个元素都在动
- 场景间的环境动效要换花样（波形 → 呼吸 → 漂移 → 扩散……）

## 四、转场类型指南

### 有限转场家族原则

> 全片只用 2-3 种转场，而且每种转场都有明确的语义。
> 转场不是装饰，是叙事的标点符号。

**为什么：**
- 转场太多样 = 花哨、不专业
- 每种转场有固定含义，观众潜意识里能理解
- 就像写文章只用句号和逗号，不用每句换一种标点

### 纳川教学视频的转场家族

| 转场类型 | 视觉效果 | 语义 | 使用频率 |
|---|---|---|---|
| **模糊叠化（blur-crossfade）** | 前一段模糊淡出，后一段模糊淡入 | 平滑过渡，"接下来……" | 80%，默认转场 |
| **上推（push-up）** | 前一段向上推出，后一段从下顶上 | 章节推进，"进入下一阶段" | ≤2 次/片，仅用于总结段前 |
| **直接切（hard cut）** | 瞬间切换 | 极快节奏、对比强烈 | 教学视频基本不用 |

### 转场时长与节奏

| 转场 | 时长 | 时机 |
|---|---|---|
| 模糊叠化 | 0.5-0.6s | 每段结尾的 0.5s 开始，下一段开头的 0.1s 完成 |
| 上推 | 0.5s | 节奏紧凑，有推进感 |

### 转场的"音频先行"原则

音频变化比视觉变化早 10-20ms。人的大脑处理声音比处理视觉快，
所以声音先变、画面后变，会感觉更自然。

**在教学视频中的应用：**
- 下一段的旁白在上一段转场开始后 0.1s 左右启动（HEAD_PAD = 0.6s，转场占 0.5s，
  剩余 0.1s 是转场结束到旁白开始的间隔）
- 如果有转场音效，音效比转场动画早 1-2 帧开始

## 五、动效编排节奏（每场景）

### 标准编排序列

```
t+0.0s  转场进行中（前一段淡出，本段淡入）
t+0.2s  场景内容开始入场（转场还剩 0.3s，内容已经"冒出来"）
t+0.3s  SVG 示意图开始绘制（描边 drawIn）
t+0.7s  示意图主体完成
t+0.9s  callout 标注弹出
t+1.2s  卡片从右侧滑入
t+1.8s  卡片内容（文字）逐行出现
t+2.5s  所有元素就位，进入"停留+环境动效"阶段
t+3.0s  旁白讲到关键信息，对应元素高亮/脉动
...
t+末-0.6s  转场开始（本段开始模糊淡出）
```

### 节奏检查清单

每场景做完后问自己：

- [ ] 入场顺序符合逻辑吗？（主体→标注→补充信息）
- [ ] 有至少 3 种不同的 ease 吗？
- [ ] 有至少 2 个不同的入场方向吗？
- [ ] 最快的动画和最慢的动画，时长差 ≥ 2 倍吗？（快慢对比才有节奏）
- [ ] 所有元素入场后，有停留时间吗？（不是一直在动）
- [ ] 环境动效有吗？是什么类型？（和上一场景不一样吧？）
- [ ] 入场时间跟旁白对齐了吗？（讲到哪，哪出现）

## 六、SVG 描边动画规范

### 描边速度

- **快速绘制**（简单图形、辅助线）：0.5-0.8s
- **正常绘制**（主体图形）：0.8-1.2s
- **慢速绘制**（强调、重要揭示）：1.2-2.0s

### 绘制顺序

就像手绘一样，按照"人会怎么画"的顺序来：

1. 先画框架/轮廓（坐标轴、容器、大形状）
2. 再画主体内容（曲线、图形）
3. 最后画标注和细节（箭头、标签、参考线）

### 虚线元素的特殊处理

**虚线不能用 drawIn** — 因为 `stroke-dashoffset` 动画会把虚线变成实线再画完，
效果不对。

正确做法：
- 虚线元素用 opacity 淡入
- 或者把虚线做成"一段一段出现"的效果（分段逐段淡入）
- 参考线、辅助线等虚线元素，建议在主体画完后再淡入

## 七、常见动效错误

| 错误 | 表现 | 修正 |
|---|---|---|
| 全 ease-out | 所有元素都是同一种缓动 | 至少 3 种 ease 对比 |
| 全 y+opacity | 所有元素都是从下往上淡入 | 至少 2 个入场方向 |
| 同时入场 | 所有元素 t+0 一起出现 | stagger 错峰，按逻辑顺序 |
| 没有停留 | 动画完了立刻转场 | 关键信息停留 1.5-2.5s |
| 环境动效太猛 | 每个元素都在大幅度动 | 幅度要小，一场景一种 |
| 线性运动 | 位置移动用 linear | 换成 power 或 sine 系列 |
| 退场动画 | 场景内元素自己做退场 | 转场就是退场，不要重复 |
| 随机 duration | 所有动画都是 0.5s | 有快有慢，形成节奏层次 |
